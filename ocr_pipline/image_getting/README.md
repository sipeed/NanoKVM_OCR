# 图像获取应用

## 项目需求

## 平台：AX630C、AX620QE（Linux）

## 功能
+ 线程 1
    + 每隔 10s 自动获取图像到系统的文件夹内：/ocr_image
    + 和上一张对比，如果相同就不保存，以及如果获取不到，获取的全是黑色，也不需要保存
    + 保存文件名格式：按照 Linux 系统的 uptime 来保存，比如
    ``` shell
    root@kvm-cccd:~# uptime
    11:20:28 up 1 day, 3 min,  1 user,  load average: 1.08, 1.06, 1.01，
    ```
    就保存为"_x_11_20_28_y_n.jpg"
        + x 表示第几次程序运行，格式 001、002 这样
        + 11_20_28 就是当前系统时间（uptime 直接获取的时间）
        + y 表示 1 day, 3 min 折合成分钟数，比如 1 day, 3 min 折合成 1443 分钟
        + n 表示相同 x 相同 y 下的第几次获取，格式 1、2 这样
+ 线程 2
    + 每隔 1s 打印一个"hello"

## 编译方法

在 `ocr_pipline` 目录下运行 build.sh：

```bash
cd /path/to/ax-npu-kit-620e/ocr_pipline
./build.sh AX630C   # 或者 AX620QE、AX620Q、AX650
```

编译完成后，可执行文件位于：
```
build_ax630c/install/bin/image_getting
```

## 使用方法

### 基本用法（使用默认摄像头 0）
```bash
./image_getting
```

### 指定摄像头索引
```bash
./image_getting 1   # 使用摄像头 1
```

## 输出说明

- 图像保存在 `/ocr_image` 目录下
- 文件名格式：`_xxx_HH_MM_SS_yyyy_n.jpg`
  - `xxx`: 程序运行次数（001, 002, ...）
  - `HH_MM_SS`: 系统运行时间的小时、分钟、秒
  - `yyyy`: 系统运行总分钟数
  - `n`: 该次运行下的第几次获取（1, 2, ...）

## 特性

1. **智能去重**: 自动比较当前图像与上一张图像，相似度超过 95% 则不保存
2. **黑屏检测**: 检测到全黑图像时自动跳过
3. **多线程**: 
   - 线程 1：每 10 秒获取并保存图像
   - 线程 2：每 1 秒打印"hello"
   - 线程 3：每 1 分钟检查文件数量，自动删除最早创建的文件以实现循环覆盖
4. **自动创建目录**: 如果 `/ocr_image` 目录不存在，会自动创建
5. **循环覆盖**: 当文件数量超过 100 个时，自动删除最早创建的文件（根据文件创建时间判断）
